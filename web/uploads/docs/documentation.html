Requerimientos

  • PHP 5.2.5 o superior con las extensiones pdo_mysql, openssl, ldap, xsl, gd2, mbstring, ctype, zip, mcrypt
  • MySQL 5.0 o superior
  • Navegador Web con cookies habilitadas.

Introducción

Este sistema gestiona planes de actividades, almacenando datos relevantes de las tareas que los componen como su Título, Tipografía, Local, Responsable, Estado, Nivel de completamiento, Periodicidad, Participantes, entre otros, que permiten obtener planificaciones más realistas con un mínimo de esfuerzo.

Actualmente el sistema es capaz de:

  • Realizar planificaciones según lo descrito en la Instrucción I y sobre la base del tiempo real disponible.
  • Generar los planes Anuales, Mensuales e Individuales de Actividades para cada entidad, departamento y/o persona de la organización.
  • Sincronizar actividades entre planes que tienen relaciones.
  • Generar Informes de Cumplimiento y Puntualizaciones de Planes de Actividades.
  • Enviar notificaciones para alertar de la necesidad de aprobación de planes Anuales y Mensuales.
  • Generar y explorar líneas bases de los planes realizados para su posterior análisis.
  • Generar gráficos y estadísticas según las necesidades de cada organización.

Instalación

  1. Descomprima el sistema en la carpeta raíz del servidor web. Si no se tiene acceso a la carpeta raíz, debe descomprimirse en un directorio de la estación de trabajo local y, después del paso 4, transferir la carpeta al servidor web usando, por ejemplo, FTP.
  2. Asegúrese que el directorio del sistema y todos sus archivos tengan el propietario adecuado (En un servidor con Linux OS, si PHP se ejecuta en modo seguro, el tener directorios y archivos con diferentes propietarios podría acarrear problemas).
  3. Configure los parámetros de su sistema editando a mano el archivo databases.yml, o usando el asistente de instalación para aquellos que prefieren la instalación gráfica.
    • Para editar manualmente el archivo, use su editor de texto preferido para ver el contenido del archivo databases.yml que se encuentra en la carpeta config del sistema de ficheros del Zentro® Instructor I. Elcontenido debe quedar como sigue: 
      all:
        doctrine:
          class: sfDoctrineDatabase
          param:
            dsn: 'mysql:host=localhost;dbname=basededatos'
            username: usuario
            password: contraseña
    • Puede usar el Asistente de Instalación. 
      INFORMACIÓN: Puede consultar video tutorial publicado aquí.        
      Cuando el sistema no puede conectarse con su base de datos; ya sea porque el servidor no está disponible, o por
una mala configuración en el archivo databases.yml automaticamente este redireccionará a la página de instalación.
  4. Abra el directorio principal del sistema en su nevegador web con lo que se debe mostrar la pantalla de bienvenida con sus respectivas opciones, o un formulario de autenticación para que se ingresen las credenciales de acceso.
